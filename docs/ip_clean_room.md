# IP Clean-Room Boundary

Gossamer is a fictional demonstrator. It must stay useful as a public portfolio artifact without exposing protected work.

## Forbidden Inputs

Do not copy or derive repository content from private project files, logs, screenshots, schemas, protocol databases, generated datasets, lab notes, acceptance procedures, host inventories, customer material, or employer material.

The following classes of information are forbidden:

- real bus databases, message IDs, packet layouts, command layouts, telemetry layouts, or transport captures,
- real spacecraft, payload, terminal, equipment, customer, program, employer, or vendor identifiers,
- real hostnames, private network addresses, serial numbers, inventory IDs, tunnel IDs, credentials, or tokens,
- real environmental chamber geometry, wiring topology, calibration tables, acceptance limits, or campaign procedures,
- real screenshots, logs, anomaly reports, test records, or historical datasets,
- prose copied from non-public internal documentation.

## Replacement Rules

Use fictional and generic concepts only:

- test article: `AuroraSat-1`,
- buses: `command_bus`, `telemetry_bus`, `subsystem_event_bus`, `facility_control_bus`,
- subsystems: `eps`, `obc`, `adcs`, `rf_comms`, `payload_sim`, `thermal_control`,
- facilities: `thermal_chamber_a`, `tvac_chamber_q1`, `vibration_table_v1`, `emc_room_e1`, `flatsat_rack_a`, `archive_node_a`,
- public-style requirement IDs such as `REQ-CYCLE-COUNT` and `REQ-DATA-QUALITY`,
- deterministic synthetic telemetry generated by this repository.

## Public Demonstration Rule

Every page, fixture, report, screenshot, and API response must be safe to publish. If a detail would require private context to justify it, replace it with a generic fictional equivalent or remove it.
